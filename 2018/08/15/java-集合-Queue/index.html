java集合-Queue

发表于 2018-08-15 | 分类于 java ， 集合

接口详情概览

Queue接口抽象的是一个FIFO队列，队列通常（但并非一定）以 FIFO（先进先出）的方式排序各个元素。不过优先级队列和 LIFO 队列（或堆栈）例外。
如下是Queue接口中方法签名信息，继承自Collection接口的方法就不再赘述了：

方法说明

offer方法

boolean offer(E e);将对象添加到队尾，添加成功返回true，失败返回false

poll方法

E poll();获取队首元素并返回（返回时移除该元素），如果队列为空，则返回null

peek方法

E peek();获取队首元素并返回（返回时不移除该元素），如果队列为空，则返回null

remove方法

E remove();获取队首元素并返回（返回时移除该元素），如果队列为空，则抛出NoSuchElementException

element方法

E element();获取队首元素并返回（返回时不移除该元素），如果队列为空，则抛出NoSuchElementException

关注点

队列提供两种插入、提取和检查操作。每个方法都存在两种形式：一种抛出异常（操作失败时），另一种返回一个特殊值（null 或 false，具体取决于操作）
