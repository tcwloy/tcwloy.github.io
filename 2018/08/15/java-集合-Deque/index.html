java集合-Deque

发表于 2018-08-15 | 分类于 java ， 集合

接口详情概览

Deque是一个线性 collection，支持在两端插入和移除元素。名称 deque 是“double ended queue（双端队列）”的缩写，通常读为“deck”。大多数 Deque 实现对于它们能够包含的元素数没有固定限制，但此接口既支持有容量限制的双端队列，也支持没有固定大小限制的双端队列。
此接口定义在双端队列两端访问元素的方法。提供插入、移除和检查元素的方法。每种方法都存在两种形式：一种形式在操作失败时抛出异常，另一种形式返回一个特殊值（null 或 false，具体取决于操作）。插入操作的后一种形式是专为使用有容量限制的 Deque 实现设计的；在大多数实现中，插入操作不能失败。
如下是Deque接口中方法签名信息，继承自Collection和Queue接口的方法就不再赘述了：

方法说明

addFirst方法

void addFirst(E e);向队首添加一个元素

addLast方法

void addLast(E e);向队尾添加一个元素

offerFirst方法

boolean offerFirst(E e);向队首添加一个元素，添加成功返回true

offerLast方法

boolean offerLast(E e);向队尾添加一个元素，添加成功返回true

removeFirst方法

E removeFirst();从队首移除一个元素并返回，如果队列为空，则抛出NoSuchElementException

removeLast方法

E removeLast();从队尾移除一个元素并返回，如果队列为空，则抛出NoSuchElementException

pollFirst方法

E pollFirst();从队首移除一个元素并返回，如果队列为空，则返回null

pollLast方法

E poolLast();从队尾移除一个元素并返回，如果队列为空，则返回null

getFirst方法

E getFirst();获取队首元素，不移除，如果队列为空，则抛出NoSuchElementException

getLast方法

E getLast();获取队尾元素，不移除，如果队列为空，则抛出NoSuchElementException

peekFirst方法

E peekFirst();获取队首元素，不移除，如果队列为空，则返回null

peekLast方法

E peekLast();获取队尾元素，不移除，如果队列为空，则返回null

removeFirstOccurrence方法

boolean removeFirstOccurrence(Object o);从此双端队列移除第一次出现的指定元素。

removeLastOccurrence方法

boolean removeLastOccurrence(Object o);从此双端队列移除最后一次出现的指定元素。

push方法

void push(E e);元素入栈，等同于addFirst方法

pop方法

void pop(E e);元素出栈，等同于removeFirst方法

descendingIterator方法

Iterator<E> descendingIterator();返回以逆向顺序在此双端队列的元素上进行迭代的迭代器。

关注点

此接口定义在双端队列两端访问元素的方法。提供插入、移除和检查元素的方法。每种方法都存在两种形式：一种形式在操作失败时抛出异常，另一种形式返回一个特殊值（null 或 false，具体取决于操作）。插入操作的后一种形式是专为使用有容量限制的 Deque 实现设计的；在大多数实现中，插入操作不能失败。

操作 抛出异常 特殊值 抛出异常 特殊值
插入 addFirst(e) offerFirst(e) addLast(e) offerLast(e)
移除 removeFirst() pollFirst() removeLast() pollLast()
检查 getFirst() peekFirst() getLast() peekLast()

此接口扩展了 Queue 接口。在将双端队列用作队列时，将得到 FIFO（先进先出）行为。将元素添加到双端队列的末尾，从双端队列的开头移除元素。从 Queue 接口继承的方法完全等效于 Deque 方法，如下表所示：

Queue 方法 等效 Deque 方法
add(e) addLast(e)
offer(e) offerLast(e)
remove() removeFirst()
poll() pollFirst()
element() getFirst()
peek() peekFirst()

双端队列也可用作 LIFO（后进先出）堆栈。应优先使用此接口而不是遗留 Stack 类。在将双端队列用作堆栈时，元素被推入双端队列的开头并从双端队列开头弹出。堆栈方法完全等效于 Deque 方法，如下表所示：

堆栈方法 等效 Deque 方法
push(e) addFirst(e)
pop() removeFirst()
peek() peekFirst()

注意，在将双端队列用作队列或堆栈时，peek 方法同样正常工作；无论哪种情况下，都从双端队列的开头抽取元素。
