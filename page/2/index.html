java集合-Deque

发表于 2018-08-15 | 分类于 java ， 集合

接口详情概览

Deque是一个线性 collection，支持在两端插入和移除元素。名称 deque 是“double ended queue（双端队列）”的缩写，通常读为“deck”。大多数 Deque 实现对于它们能够包含的元素数没有固定限制，但此接口既支持有容量限制的双端队列，也支持没有固定大小限制的双端队列。
此接口定义在双端队列两端访问元素的方法。提供插入、移除和检查元素的方法。每种方法都存在两种形式：一种形式在操作失败时抛出异常，另一种形式返回一个特殊值（null 或 false，具体取决于操作）。插入操作的后一种形式是专为使用有容量限制的 Deque 实现设计的；在大多数实现中，插入操作不能失败。

关注点

此接口定义在双端队列两端访问元素的方法。提供插入、移除和检查元素的方法。每种方法都存在两种形式：一种形式在操作失败时抛出异常，另一种形式返回一个特殊值（null 或 false，具体取决于操作）。插入操作的后一种形式是专为使用有容量限制的 Deque 实现设计的；在大多数实现中，插入操作不能失败。

此接口扩展了 Queue 接口。在将双端队列用作队列时，将得到 FIFO（先进先出）行为。将元素添加到双端队列的末尾，从双端队列的开头移除元素。从 Queue 接口继承的方法完全等效于 Deque 方法，如下表所示：

Queue 方法 等效 Deque 方法
add(e) addLast(e)
offer(e) offerLast(e)
remove() removeFirst()
poll() pollFirst()
element() getFirst()
peek() peekFirst()

双端队列也可用作 LIFO（后进先出）堆栈。应优先使用此接口而不是遗留 Stack 类。在将双端队列用作堆栈时，元素被推入双端队列的开头并从双端队列开头弹出。堆栈方法完全等效于 Deque 方法，如下表所示：

堆栈方法 等效 Deque 方法
push(e) addFirst(e)
pop() removeFirst()
peek() peekFirst()

注意，在将双端队列用作队列或堆栈时，peek 方法同样正常工作；无论哪种情况下，都从双端队列的开头抽取元素。

集合是什么

集合是容纳对象的容器。我们在写代码的时候经常会遇到这种场景，需要一个东西来容纳多个对象,传统方式会通过数组来满足这个需求，然而数组有其局限性（数组初始化时就需要指定大小），当我们不确定容纳对象的个数时，数组就无法满足了，这个时候，java标准库中提供的集合是更好的选择。
集合类主要负责保存、盛装其他数据，因此集合类也被称为容器类。所以的集合类都位于java.util包下，后来为了处理多线程环境下的并发安全问题，java5还在java.util.concurrent包下提供了一些多线程支持的集合类。

集合继承体系

对于上图做如下说明：

  • Collection接口继承自Iterable，图中并未画出
  • Deque（双端队列）接口继承自Queue,图中未画出
  • Stack、Vector、HashTable为遗留容器，尽量避免使用
  • List经常用来存储有序序列（有序指的是存储顺序），常用实现类为ArrayList和LinkedList
  • Set经常用来去除重复数据，常用实现类是HashSet，如果需要有序则使用TreeSet（需要排序规则）
  • Map经常用来保存键值对，常用实现类是HashMap，如果需要有序则使用TreeMap（需要排序规则）
  • Comparable与Comparator：可以把Comparable理解为内部比较器，而Comparator是外部比较器,实现Comparator接口代码更加灵活，可以定义某个类的多个比较器，从而在排序时根据实际场景自由调用，而Comparable接口实现后便不能改动
  • Collections和Arrays是两个工具类，用法后面单独介绍

JVM是什么

JVM是Java Virtual Machine（Java虚拟机）的缩写，JVM包含在JRE中，是java能够跨平台的基础，JVM屏蔽了底层操作系统的细节，提供了java字节码的运行环境，所有java程序都会被编译成.class也就是字节码文件，字节码文件先由我们的JVM进行解释，最后再交给本地系统，实现了Java的可移植性。
